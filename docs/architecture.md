# System Architecture: Adaptive AI Enterprise Council

## Overview
Five AI agents representing C-suite roles independently evaluate financial risk,
then vote via credibility-weighted aggregation. All reasoning is generated by
locally hosted LLMs via Ollama — no external API calls, no cloud dependency.

## Agent Design & Model Assignment

| Agent | Focus | Risk Threshold | Reasoning Model | Key Bias |
|-------|-------|---------------|----------------|---------|
| CEO | Strategic growth | 0.55 | llama3 | Growth potential 40% |
| CFO | Capital stability | 0.35 | deepseek-r1 | Liquidity risk 35% |
| Marketing | Market expansion | 0.60 | mixtral | Market opportunity 40% |
| PR | Reputation risk | 0.45 | llama3 | Public sentiment 35% |
| Legal | Regulatory compliance | 0.30 | deepseek-r1 | Regulatory risk 40% |

## Why These Model Assignments?
- **deepseek-r1** for CFO and Legal: strong chain-of-thought reasoning for numerical and rule-based analysis
- **llama3** for CEO and PR: well-rounded generation for strategic and stakeholder-aware language
- **mixtral** for Marketing: mixture-of-experts diversity matches the creative/multi-angle nature of market analysis
- **phi** is configured as fallback for any missing model

## LLM Routing (LocalLLMClient)
All agents share one `LocalLLMClient` instance that talks to `http://localhost:11434`.
The client resolves role → model tag and POSTs to `/api/generate`.
Graceful fallback: if a model is unavailable, the client returns a stub reasoning string
and logs a warning — the pipeline does not crash.

## Credibility Formula
```
Credibility(t+1) = α·C(t) + β·Performance + γ·Agreement − δ·HistoricalError
Default: α=0.7, β=0.15, γ=0.10, δ=0.05
```

## Decision Aggregation
```
WeightedRisk = Σ(Credibility_i × RiskScore_i) / Σ(Credibility_i)
< 0.33 → APPROVE   |   0.33–0.66 → CONDITIONAL_APPROVE   |   > 0.66 → REJECT
```

## Evolution Mechanism
- Agent monitored over N cycles (default: 3)
- Lowest credibility for 3 consecutive cycles → agent removed
- Replacement = mutated offspring of highest-credibility agent
- Mutation: ±10% weight perturbation (normalized), ±5% threshold drift
- Replacement agent starts at 0.5 credibility

## Technology Stack
- ML Base Model: XGBoost (shared across agents)
- LLM Reasoning: Ollama (llama3 · deepseek-r1 · mixtral · phi)
- API: FastAPI + Pydantic
- Config: YAML-driven (no magic numbers in code)
- Tests: pytest
